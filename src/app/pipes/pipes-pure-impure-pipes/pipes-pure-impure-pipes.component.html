Pure and Impure Pipes

Pure Pipes:

  • Re-compute the output only when the input changes.
  • They are optimized for performance.

    • Impure Pipes:

  • Re-compute the output for every change detection cycle.
  • Useful when dealing with data that changes frequently, like arrays or objects.

    • To create an impure pipe, set pure: false in the @Pipe decorator.

    	@Pipe({
		name: 'kebabCase',
		pure: false
	})

    By default, pipes are pure.

    Avoid creating impure pipes unless absolutely necessary, as they can incur a significant performance penalty if used without care.
